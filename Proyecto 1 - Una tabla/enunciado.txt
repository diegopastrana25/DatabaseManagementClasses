PROYECTO 1
Una tabla
Hasta la fecha hemos aprendido a crear una sola tabla en una base de datos pero lo hemos
hecho casi siempre siguiendo un “guión dado” en esta ocasión vamos a crear una tabla, pero
partiendo de la descripción de un problema:
Tenemos como cliente un hotel, el Hotel ICSE que desea almacenar información de los
huéspedes que se hospedan en el complejo. Esta información es la siguiente:
- Nombre.
- Apellidos.
- Fecha de nacimiento.
- Sexo.
- Número de teléfono.
- DNI.
- Dirección completa.
- Código postal.
- Si requiere plaza de aparcamiento, o no.
- Comentarios, sugerencias o quejas.
- Tipo de habitación: individual, doble o familiar.
- El número de habitación, solo hay 6: 1.1, 1.2, 1.3, 2.1, 2.2, 2.3
- El método de pago: Efectivo o tarjeta.
- Fecha de entrada.
- Fecha de salida.
- Número de la reserva.
Un cliente puede hacer todas las reservas que quiera incluso durante las mismas fechas, pero,
una habitación, no puede estar reservada por más de una persona en el mismo momento.
La tabla la haremos en una base de datos nueva llamada ICSEHotelDB y se llamará customer.
Es importante recordar que debemos generar una base de datos lo más consistente posible
por lo que es importante que dejemos claras restricciones obvias.
Hay que hacer el SQL de la base de datos desde cero sin utilizar la interfaz de phpMyAdmin,
es decir, mediante sentencias.
Se valorará:
- La correcta estructura del código y el buen nombrado de los elementos.
- La funcionalidad de la tabla.
- La correcta selección de los tipos de datos.La entrega de esta tarea se realizará vía Moodle y mediante un documento .sql con los
comandos utilizados y con el siguiente formato de nombre:
Proyecto1_Nombre_Apellido1_Apellido2.sql
Ejemplo:
Proyecto1_Hadriel_Fernando_Quintero_Ceballos.sql
